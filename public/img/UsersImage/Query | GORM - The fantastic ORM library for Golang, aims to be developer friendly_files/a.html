                                      Song Lusic • 5 months ago • edited

                                      There is a bug in the doc. db.First(&user, 10) is NOT "SELECT * FROM users WHERE id = 10;" . The correct SQL for this code is "SELECT * FROM users WHERE (id = 10) ORDER BY id ASC LIMIT 1".

                                      **Edited1:** Not only one bug. For instance: `db.Where("name = ?", "jinzhu").First(&user)` and more. All queries with `First` will have "ORDER BY id ASC" in query, but this doc omits this important part in many places. This should not be omitted because with or without it are two completely different queries. The doc should either show correct equivalent SQL codes or not show the SQL code at all. But showing the wrong SQL code should be avoided.

                                      **Edited2:** I found more bugs on sql comments. I will not list them all here, because too many. For instance, db.Not("name = ?", "jinzhu").First(&user) should have LIMIT 1, but the sql comment selects all.
